1. Compiler คืออะไร?
Compiler คือโปรแกรมที่ทำหน้าที่แปลง โค้ดต้นฉบับ (Source code) ที่เราเขียนในภาษาระดับสูง เช่น C, C++, Java ฯลฯ
ให้กลายเป็น โค้ดเครื่อง (Machine code) หรือไฟล์โปรแกรมที่คอมพิวเตอร์สามารถรันได้โดยตรง (เช่นไฟล์ .exe)

การแปลงนี้เกิดขึ้น ก่อนการรันโปรแกรม (Compile-time)

เมื่อ compile เสร็จ เราจะได้ไฟล์โปรแกรมที่พร้อมรันได้เลย โดยไม่ต้องแปลซ้ำอีก

ข้อดีของ Compiler
โปรแกรมที่ได้จะรันได้เร็ว เพราะแปลงเป็นโค้ดเครื่องไว้ล่วงหน้าแล้ว

สามารถตรวจจับข้อผิดพลาดหลายอย่างได้ตอน compile ก่อนรัน

เหมาะกับโปรแกรมที่เน้นประสิทธิภาพสูง

ข้อเสียของ Compiler
ต้อง compile ก่อนทุกครั้งถ้าโค้ดมีการเปลี่ยนแปลง

เวลาแก้ไขโค้ด ต้องรอ compile ใหม่ก่อนรัน

2. Interpreter คืออะไร?
Interpreter คือโปรแกรมที่ทำหน้าที่อ่านและแปล โค้ดต้นฉบับ ทีละบรรทัด แล้วสั่งให้คอมพิวเตอร์ทำงานทันที (Run-time)

ไม่มีการแปลงโค้ดเป็นไฟล์ executable ล่วงหน้าเหมือน Compiler

ตัวอย่างภาษาโปรแกรมที่ใช้ Interpreter เช่น Python, JavaScript (ในบางกรณี)

ข้อดีของ Interpreter
สะดวกและเร็วในการทดสอบโค้ด เพราะรันได้ทันทีโดยไม่ต้อง compile

เหมาะกับการพัฒนาโปรแกรมขนาดเล็ก หรือการเรียนรู้ภาษาโปรแกรม

แก้ไขโค้ดแล้วรันได้ทันที เหมาะกับการ debug

ข้อเสียของ Interpreter
โปรแกรมที่รันจะช้ากว่า เพราะแปลทีละบรรทัดทุกครั้ง

การตรวจจับข้อผิดพลาดจะเกิดขึ้นตอนรัน ทำให้เจอ error ทีละบรรทัด
